# Команды, разделы и логика 𐌽ꤕ𐌗ተ

Документ описывает, что делает каждая кнопка/команда и как это устроено внутри `bot.py`.

---

## 1. Команды (`/`)

| Команда | Что делает | Где в коде |
|---|---|---|
| `/start` | Регистрирует пользователя, показывает приветствие и главное меню | `cmd_start` |
| `/start <код>` | Deep-link на чужую анон-ссылку → выбор «Вопрос / Валентинка» | `cmd_start` → `handle_incoming_link` |
| `/start ref_<id>` | Реферальный deep-link → начисляет коины пригласившему | `cmd_start` → `handle_referral` |
| `/id` | (скрытая) Показать свой Telegram ID | `_h_cmd_id` |
| `/me` | (скрытая) Открыть свой профиль | `_h_cmd_me` |
| `/admin` | (скрытая, только админ) Открыть админ-панель | `_h_cmd_admin` |
| `/moder` | (скрытая, только персонал) Открыть панель модератора | `_h_cmd_moder` |
| `/userinfo` | (скрытая, только персонал) Карточка пользователя по ID | `_h_cmd_userinfo` |

> Скрытые команды НЕ отображаются в меню Telegram (в `set_my_commands` только `/start`), но работают при вводе.

Меню целиком управляется reply-кнопками — отдельных слэш-команд больше нет.

## 2. Главное меню

| Кнопка | Раздел | Логика |
|---|---|---|
| 🔗 Моя ссылка | Личная ссылка | `show_link_menu`, `show_my_link`, `process_link_code` |
| 🎲 Чат-рулетка | Поиск собеседника | `roulette_*`, фоновый `_matchmaker_loop` |
| 👤 Профиль | Профиль, баланс, VIP | `show_profile` |
| 🛒 Магазин | Покупки за коины/Stars | `show_shop`, `do_purchase` |
| 👥 Пригласить | Рефералы и награды | `show_referral` |
| ℹ️ Помощь | Справка | `show_help` |
| 🌐 Язык | Смена языка | `show_language_menu` |

## 3. Анонимка по ссылке

1. Пользователь создаёт код в «Моя ссылка» → `process_link_code`.
2. Другой человек открывает `t.me/bot?start=КОД` → `handle_incoming_link` показывает выбор типа.
3. Выбор «❓ Вопрос / 💌 Валентинка» → `on_anon_type_text`.
4. Текст/голос/медиа → `process_anon_content` → доставка адресату через `deliver_anon`.
5. Получатель видит инлайн-кнопки: **Ответить**, **Жалоба**, **Узнать кто**.

**Анти-спам:** `has_forbidden_contacts` блокирует ссылки, @юзернеймы, номера, соцсети (персонал — исключение).

**Устойчивость к рестарту:** незавершённый флоу хранится в таблице `link_flow` (`save_link_flow` / `load_link_flow`).

**Лимиты:** до `DAILY_LIMIT` (20) отправок в сутки для не-VIP; ответы — без лимита.

## 4. Награды за рефералов

Настраиваются админом (кнопка **✏️ Изменить** в «Пригласить»), хранятся в таблице `settings`:

| Награда | По умолчанию | Логика |
|---|---|---|
| 💎 Коины за друга | 20 (VIP — 50) | `handle_referral` |
| 🎁 Бесплатный VIP | 5 друзей → 7 дней | `on_claim_vip`, `cfg_vip_*` |
| 🛡 Модерка на неделю | 10 друзей → 7 дней | `on_claim_moder`, `cfg_moder_*` |
| 💎 Бонус за активность | +20 за каждые 10 отправок/ответов по ссылке | `reward_link_activity` |

Засчитываются только друзья, **создавшие свою ссылку** (`qualified_referrals`). VIP и персонал не получают бонус за активность. Временная модерка хранится в `users.moder_until` и сама истекает.

---

## 5. Раскрытие отправителя (Telegram Stars)

- Кнопка «Узнать кто» → `on_reveal_button` → подтверждение → `on_reveal_pay`.
- Отправляется инвойс на **1 ⭐** (`currency="XTR"`), оплата проходит через Telegram.
- `on_precheckout` подтверждает, `on_successful_payment` раскрывает данные.
- Админ/модер раскрывают **бесплатно и мгновенно** (`is_unlimited`).

---

## 6. Роли

| Роль | Как определяется | Привилегии |
|---|---|---|
| Админ | `tg_id` в `ADMIN_IDS` | Всё без ограничений, бесконечные коины |
| Модер | `users.is_moder` или активный `users.moder_until` | VIP навсегда, безлимит, модерация |
| VIP | активный `users.vip_until` | Скидки, без лимитов, ежедневный бонус |

Хелперы: `is_admin`, `is_moder`, `is_vip`, `is_unlimited`, `is_staff`.

---

## 7. База данных

Основные таблицы (`init_db`): `users`, `anon_messages`, `reports`, `bans`, `roulette_queue`,
`roulette_sessions`, `referrals`, `link_flow`, `settings`, `shop_items`, `purchases`,
`moder_apps`, `star_packages`, `mandatory_channels`.

Совместима с **SQLite** и **PostgreSQL** через слой `_PgConnection` / `_PgCursor`. Недостающие
колонки добавляются безопасно в `migrate()`.
