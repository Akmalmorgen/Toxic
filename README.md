# 𐌽ꤕ𐌗ተ — анонимный Telegram-бот

Бот для **анонимного общения**: вопросы и валентинки по личной ссылке, чат-рулетка по полу, магазин, рефералы с наградами и мультиязычность (🇷🇺 / 🇺🇿 / 🇬🇧).

Написан на [aiogram 3](https://docs.aiogram.dev/), работает на **PostgreSQL** (рекомендуется) или **SQLite**. Запускается на любом сервере — Render, Railway, VPS или в Docker.

---

## ✨ Возможности

| Раздел | Что умеет |
|---|---|
| 🔗 **Моя ссылка** | Личная ссылка `t.me/bot?start=КОД`. Люди пишут тебе анонимно: ❓ вопрос или 💌 валентинка. Кнопка «Поделиться» с превью-карточкой бота. |
| 🎲 **Чат-рулетка** | Случайный собеседник по полу. Управление снизу: «Далее» / «Стоп». |
| 👁 **Узнать кто** | Раскрытие отправителя анонимки за **реальные Telegram Stars** ⭐. |
| 👥 **Пригласить** | Рефералы: коины за друзей + **бесплатный VIP** и **модерка на неделю** за приглашённых. Настройки наград редактируются админом. |
| 🛒 **Магазин** | Покупка коинов, VIP, модерки за коины и за Telegram Stars. |
| 🛡 **Модерация / Админка** | Жалобы, баны, рассылки, обязательные каналы, статистика, выдача модерки/коинов. |
| 🛡 **Анти-спам** | Блокировка ссылок, @юзернеймов, номеров и упоминаний соцсетей в анонимках. |
| 🌐 **Языки** | Русский, узбекский, английский. |

## 🚀 Быстрый старт (локально)

```bash
git clone https://github.com/Akmalmorgen/Toxic.git
cd Toxic

cp .env.example .env          # впишите BOT_TOKEN и ADMIN_IDS
pip install -r requirements.txt
python main.py
```

Токен бота берётся у [@BotFather](https://t.me/BotFather), свой Telegram ID — у [@userinfobot](https://t.me/userinfobot).

## 🔧 Переменные окружения

| Переменная | Обязательна | Описание |
|---|---|---|
| `BOT_TOKEN` | ✅ | Токен бота от @BotFather |
| `ADMIN_IDS` | ✅ | ID администраторов через запятую: `123,456` |
| `DATABASE_URL` | ⚠️ рекомендуется | Строка PostgreSQL (например, [Neon](https://neon.tech)). Без неё — SQLite, который **стирается при перезапуске** на облачных хостингах. |
| `DB_PATH` | — | Путь к SQLite-файлу (если нет `DATABASE_URL`). По умолчанию `./bot.db` |

## ☁️ Деплой

### Render (бесплатно)
1. New → **Web Service** → подключите этот репозиторий.
2. Render сам прочитает [`render.yaml`](render.yaml).
3. В **Environment** задайте `BOT_TOKEN`, `ADMIN_IDS`, `DATABASE_URL`.
4. Deploy. Keep-alive сервер держит порт открытым; для бесплатного плана можно пинговать URL через [UptimeRobot](https://uptimerobot.com), чтобы сервис не засыпал.

### Railway / Heroku
Используется [`Procfile`](Procfile): `web: python main.py`. Задайте те же переменные окружения.

### Docker (на любом VPS)
```bash
docker compose up -d --build      # запустит бота (и опционально Postgres)
docker compose logs -f            # логи
```
Перед этим заполните `.env`. Подробнее — [`docker-compose.yml`](docker-compose.yml).

### Обычный VPS (systemd)
```bash
pip install -r requirements.txt
python main.py
```
Рекомендуется обернуть в systemd-сервис или `screen`/`tmux` для автозапуска.

## 🗂 Структура проекта

```
Toxic/
├── bot.py              # основной код бота (хэндлеры, БД, логика)
├── main.py             # точка входа (запускает bot.main())
├── requirements.txt    # зависимости Python
├── .python-version     # версия Python (3.12)
├── .env.example        # шаблон переменных окружения
├── Procfile            # запуск для Railway/Heroku
├── render.yaml         # конфигурация Render
├── Dockerfile          # образ для Docker
├── docker-compose.yml  # запуск в Docker (+ опц. Postgres)
├── assets/             # аватар бота и скрипт его генерации
└── docs/
    └── COMMANDS.md     # команды, разделы меню и логика по полочкам
```

> Бот реализован как **один файл `bot.py`** — это сознательный выбор: проще деплоить и поддерживать в одном месте. Описание команд и логики — в [`docs/COMMANDS.md`](docs/COMMANDS.md).

## 🤖 Команды

| Команда | Действие |
|---|---|
| `/start` | Запуск, главное меню |
| `/start <код>` | Открыть анонимку по чужой ссылке |
| `/start ref_<id>` | Зайти по реферальной ссылке |

Всё остальное управляется кнопками. Полный разбор — в [`docs/COMMANDS.md`](docs/COMMANDS.md).

## 🔐 Безопасность

- **Не коммитьте реальный `BOT_TOKEN`** в публичный репозиторий. Используйте переменные окружения на хостинге, а `.env` держите локально (он в `.gitignore`).
- Если токен попал в историю git — отзовите его у @BotFather и создайте новый.
